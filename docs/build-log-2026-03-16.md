## Bugs Fixed

### 1. SPIN Sales Coach — Wrong Agent Prompt After Credit Outage
**Symptom**: Marcus Webb was asking about "sales scenarios" instead of using the SPIN framework. He did not know the consultant's name.

**Root cause**: Commit `3b8863b` had switched `streamCoachResponse()` to use `agent.prompt` from the database as the system prompt. The SPIN agent's `agent.prompt` was a thin generic string that didn't include the Marcus Webb persona, dynamic name injection, or session context.

**Fix**: Reverted `3b8863b`. The SPIN Sales Coach system prompt is now hardcoded in the `agentConfig` template inside `lib/coaching.ts` — this is intentional and permanent. The template dynamically injects `preferredName`, `resumeText`, `company`, `interviewType`, and RAG context at runtime.

**Architecture decision**: 
- SPIN Sales Coach → hardcoded `agentConfig` template (never use `agent.prompt`)
- All other agents (Assessment, future) → `agent.prompt` from database, with session context and RAG appended

**Documented in**: CLAUDE.md — "SPIN Sales Coach — System Prompt is Hardcoded"

---

### 2. Assessment Agent — Asking About Sales Instead of AI Assessment
**Symptom**: Assessment agent opened sessions asking about upcoming sales conversations.

**Root cause (layered)**:
1. `streamCoachResponse()` was using `agent.prompt` but the Assessment session context stored wrong defaults (`role: "Software Engineer"`, `interviewType: "General"`) from Interview Coach App era
2. Non-SPIN agents had SPIN-specific fields (`role`, `interviewType`, `stage`) appended to their prompts

**Fix**:
- `/api/elevenlabs-signed-url` now stores Assessment-specific context when `agent_type === 'assessment'`: `role: 'Consultant'`, `interviewType: 'AI Assessment Learning'`
- `streamCoachResponse()` for non-SPIN agents only appends `preferredName`, `resumeText`, and RAG context — no SPIN-specific fields

---

### 3. Generate Scorecard Button — Transcript Not Ready Error
**Symptom**: Clicking "End Session & Generate Scorecard" without stopping voice first caused a "Conversation transcript is not ready yet" error. Agent kept talking.

**Root cause**: ElevenLabs only finalizes the transcript after the conversation ends. Clicking the button while voice was still active triggered the transcript fetch too early.

**Fix**: 
- Button renamed to "Generate Scorecard"
- Button disabled until user clicks "End Session" first
- `handleGoToScorecard` now calls `conversation.endSession()` before transcript fetch, with 1500ms delay

**Documented in**: CLAUDE.md — "Transcript Not Ready Error"

---

### 4. Voice LLM — Session Context Cross-Contamination
**Symptom**: Running a SPIN session after an Assessment session (or vice versa) caused the wrong agent persona to be used.

**Root cause**: `getLatestSessionContext()` fetches the most recent session from `voice_sessions` by `created_at`. If the wrong agent's session ran more recently, its context was returned.

**Status**: Partially mitigated. The signed URL route now stores `agentId` in context and `streamCoachResponse()` uses `coachId` to look up the correct agent. Full fix (sessionId or conversationId-based lookup) is documented as a future improvement.

---

### 5. Anthropic API Credit Outage
**Symptom**: Voice agent connected but went silent after first user turn. Logs showed `400 - credit balance too low`.

**Fix**: Added credits at console.anthropic.com. No code changes required.

## Features Built

### Live Research Capability (Linear ticket 39) — Foundation Only

Built the infrastructure for per-agent live company research. Not wired into any session flow yet — the geopolitical analysis agent is the intended primary use case.

**What was built:**
- `app/api/research/company/route.ts` — POST `{ companyName }` → returns `{ companyName, brief, retrievedAt }`. Uses Anthropic with `web_search_20250305` tool. Timeout: 40 seconds.
- `live_research_enabled` boolean column on `agents` table (default false)
- Prisma schema updated
- Migration: `supabase/migrations/20260316000000_add_live_research_to_agents.sql`

**What was NOT built:**
- Session page wiring
- VoiceCoach integration
- Coaching prompt injection

**Key decision**: Live Research toggle removed from Prompt Control UI. This is a dev-time decision per agent, not an admin toggle. A 40 second research call has major UX and cost implications that require purpose-built experience design — not a simple on/off switch.

**Tested**: Research API confirmed working via curl on production. Holman Logistics brief returned real, specific data (1,500 employees, 8M sq ft warehousing, key clients, competitive landscape).

## Architecture Decisions

| Decision | Rationale |
|----------|-----------|
| SPIN system prompt hardcoded in `agentConfig` | Dynamic context injection (name, company, resume) requires code-level template. `agent.prompt` can't do this without variable substitution plumbing. |
| Non-SPIN agents use `agent.prompt` from database | Prompt Control is the right place to manage learning/coaching agent behavior. No dynamic context needed beyond name and RAG. |
| Live research is dev-time config, not UI toggle | 40s latency and API cost require purpose-built UX. Wrong tool for an admin switch. |
| Geopolitical agent is primary live research use case | Deep research with async job pattern and results display is the right architecture. Sales coaching is a nice-to-have, not a core need. |
